🔒 Security Layer
Active Protections
This report runs with multiple defense layers to keep your Manager.io data safe. Every check is enforced at runtime.
✓
Zero external tracking NO ANALYTICS
No telemetry, no analytics, no cookies, no fingerprinting. Your sales data never leaves the Manager.io iframe boundary.
✓
Origin-pinned postMessage ACTIVE
After the first successful response, the host origin is locked. Any message from a different origin is silently rejected — a malicious iframe can't spoof API responses.
Locked origin: (not yet established — click Load to connect)
✓
Read-only API access GET ONLY
Only HTTP GET requests are ever made — specifically to the sales-invoice-batch endpoint. No writes, no mutations, nothing you do here can change data.
✓
Endpoint whitelist ENFORCED
Every outgoing path is validated against a strict allow-list. Only the expected sales-invoice endpoint can be requested — no rogue calls possible, even if the API response tries to inject a malicious next-page URL.
✓
Content Security Policy LOCKED
A strict CSP header blocks all external network calls and remote scripts, except the Chart.js library (loaded from cdnjs). The page talks to Manager.io only through its sandboxed postMessage bridge.
✓
XSS-safe rendering ESCAPED
Error messages and any dynamic text from API responses are HTML-escaped before rendering, so hostile data inside a record can't execute scripts in the page.
All checks run client-side, in your browser v2.0
